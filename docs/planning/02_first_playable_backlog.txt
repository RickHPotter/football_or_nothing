Football or Nothing - First Playable Backlog
============================================

Goal
----
Reach a minimal playable football-management loop before expanding the world.


Phase 1 - Foundation
--------------------
Status: mostly implemented.

1. Add default Rails authentication.
2. Create the user-to-manager career concept.
3. Create countries.
4. Create clubs.
5. Create initial club finances.
6. Create stadiums.
7. Create athletes.
8. Create athlete attributes.
9. Create athlete contracts.
10. Generate a tiny fictional world:
   - 1 country
   - 8 clubs
   - 8 stadiums
   - 18 to 25 athletes per club
   - Basic financial profiles for each club
   - 1 manager controlled by the user


Phase 2 - Club Management Skeleton
----------------------------------
Status: mostly implemented.

1. Show manager dashboard.
2. Show current club dashboard.
3. Show squad list.
4. Show athlete profile.
5. Show contracts and wages.
6. Allow manager to choose or be assigned to a club.
7. Show club financial summary.
8. Prevent the player from acting as club owner.


Phase 3 - Tournament Skeleton
-----------------------------
Status: implemented for a generated domestic league.

1. Create tournament.
2. Create tournament edition.
3. Register clubs through tournament participations.
4. Generate fixtures for a simple league format.
5. Store standings.
6. Mark champion at season end.


Phase 4 - Match Simulation v1
-----------------------------
Status: implemented.

Implemented:
- Fixture page.
- Deterministic instant simulation.
- Match status and score storage.
- Standings updates.
- Match timeline with goal events.
- Career date advancement to match day and next match.
- Basic athlete stats from simulated matches.
- Lineups and bench selections.
- Live clock in 15-minute advances.
- Pausing and resuming for manager decisions.
- Substitutions.
- Basic tactical changes through formation and mentality.

Deferred:
- True minute-by-minute simulation events.
- Tactical changes influencing simulation outcomes.


Phase 5 - History
-----------------
Status: core implemented.

Implemented:
- Store trophies.
- Store tournament participation outcomes.
- Complete tournament editions when all fixtures are completed.
- Assign champion from standings.
- Store manager season stats.
- Store club season stats.
- Store athlete season stats.
- Show trophy history on career and club pages.
- Show manager season history on career page.
- Show club season history and top scorers on club page.
- Show athlete stats on athlete profiles.

Deferred:
- Top scorer awards.
- Best player/keeper/young player awards.
- News/history feed.
- Hall of fame records.
- Promotion/relegation history.
- Rich financial season snapshots.


Phase 6 - Transfer Skeleton
---------------------------
Status: implemented.

Implemented:
- Create transfers as historical records.
- Support free transfer.
- Support permanent transfer.
- Update athlete contracts through transfer flow.
- Keep old contracts/history intact.
- Enforce basic transfer budget and wage budget rules.
- Show a transfer-market screen for the current club.

Deferred:
- Loans.
- Release clauses as enforced selling rules.
- Transfer windows.
- AI club transfer activity.


Phase 7 - RPG Progression
-------------------------
Status: implemented.

Implemented:
- Add manager reputation.
- Add athlete attributes and potential.
- Add age-based ability progression and decline.
- Add morale and condition.
- Add club reputation.
- Use manager reputation to limit available jobs.
- Apply progression once when a tournament edition is finalized.

Deferred:
- Training plans.
- Detailed development curves.
- Injuries and fatigue.
- Player personality.
- Dynamic club reputation beyond champion gains.


Phase 8 - Season Rollover
-------------------------
Status: implemented.

Implemented:
- Create the next tournament edition from a completed edition.
- Carry participating clubs forward.
- Create fresh tournament participations with reset standings.
- Generate new fixtures with the existing league scheduler.
- Preserve previous season history.
- Expose a career-page action when no upcoming fixture exists and a completed
  season can be rolled over.
- Advance the career date to the first fixture of the new season.
- Avoid duplicate next seasons when rollover is triggered repeatedly.

Deferred:
- Promotion and relegation.
- Transfer windows.
- Contract expiration consequences.
- AI club squad changes.
- Calendar conflicts.
- Cup and international competition rollover.
- Season financial accounting.
- Full preseason.


Phase 9 - Frontend Playable UI Pass
-----------------------------------
Status: implemented.

Implemented:
- Add Tailwind CSS build pipeline.
- Add a compact game layout and top navigation.
- Add shared utility classes for panels, stat tiles, tables, badges, buttons,
  flashes, and forms.
- Redesign home, sign in, registration, career creation, career dashboard,
  club dashboard, fixture/match screen, transfer market, and athlete profile.
- Keep the UI dense and data-first, closer to a football management game than a
  corporate admin panel.

Deferred:
- Full visual design system.
- Responsive mobile polishing beyond basic layout behavior.
- Icons and richer interaction affordances.
- Dedicated history/news screen.
- Advanced match visualizations.


Phase 10 - Match Event Depth
----------------------------
Status: implemented.

Implemented:
- Add detailed event types for major chances, yellow cards, red cards,
  injuries, and substitutions.
- Generate deterministic non-goal timeline events during match simulation.
- Keep goal events and score storage deterministic.
- Store card and injury totals on athlete season stats.
- Show detailed event labels in the match timeline.
- Show cards and injuries on athlete stat tables.

Deferred:
- True possession-by-possession simulation.
- Tactical settings influencing event probabilities.
- Player condition impact during the match.
- Suspensions and injury availability rules.
- Assists and ratings generated from events.


Phase 11 - Transfer Offers
--------------------------
Status: implemented.

Implemented:
- Add transfer offers as negotiation records.
- Submit pending offers from the transfer market.
- Track offered fee, wage, source club, target club, expiry, and status.
- Finalize acceptable pending offers into completed transfers.
- Keep recent offers visible alongside completed transfer history.
- Preserve transfer budget and wage validation through the existing transfer
  processor.

Deferred:
- Selling-club AI decision making.
- Counter offers.
- Loan offers.
- Transfer windows.
- Contract release clauses as hard selling rules.
- Offer expiry automation.


Important Deferrals
-------------------
These are important, but not required for the first playable loop:

- Complex tactics.
- Detailed match events.
- Full staff model.
- Youth academy.
- Scouting.
- Sponsorships.
- Stadium construction and upgrades.
- International tournaments.
- National teams.
- Multi-country calendar.
- Advanced financial rules.
- Press/news system.
- Save import/export.
- Real data import.
